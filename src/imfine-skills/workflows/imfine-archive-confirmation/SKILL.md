# imfine Archive Confirmation

Use this workflow to confirm final gates, archive evidence, commit/push policy, and project knowledge updates.

This workflow absorbs Superpowers `verification-before-completion` and `finishing-a-development-branch`, then adapts them to imfine's Archive Agent plus runtime-finalize split.

## Iron Law

No completion claim without fresh verification evidence.

Before any run is described as complete, archived, pushed, or demo-ready:

1. Identify the runtime artifact or command that proves the claim.
2. Read or run the full check freshly.
3. Inspect the exit code or gate status.
4. Report actual status, including blockers.
5. Only then allow Archive Agent judgment or runtime finalization.

## Checks

- Runtime final gates are generated by `imfine-runtime`.
- Provider receipts close required native Agent contracts.
- QA and Review evidence are present.
- Accepted deviations are documented.
- Archive report and final report are present.
- Sandbox verification is present when required by runtime policy.
- Runtime requirements and QA environment evidence do not contradict each other.
- Project knowledge freshness is current or explicitly blocked.
- Commit/push policy is satisfied or exposed as a blocker.

## Archive Flow

1. Verify QA, Review, Risk, Committer, true harness, and project knowledge gates.
2. Dispatch `imfine-agent-archive` for Agent-authored archive judgment.
3. Require provider-origin completed receipt for Archive Agent.
4. Run runtime archive finalization only after Archive Agent handoff exists.
5. If finalization blocks, write structured blockers and do not claim completion.
6. If push is blocked, report `ready_for_commit`, `awaiting_user_approval`, or `blocked`; do not report completed.

## Completion Evidence Table

| Claim | Required Evidence |
| --- | --- |
| Tests pass | QA command output and `evidence/test-results.md` |
| Review approved | Reviewer handoff and `evidence/review.md` |
| Native Agents ran | provider-origin completed receipts with valid output snapshots |
| Final gates pass | runtime-generated `orchestration/final-gates.json` |
| True harness passed | fresh `orchestration/true-harness-evidence.json` |
| Archive complete | Archive Agent handoff plus runtime archive finalization |

## Rules

- Do not archive misleading or incomplete runs.
- Do not treat root-level hand-written gates as authoritative.
- Surface follow-up items clearly.
- Do not let runtime-only receipts prove Agent execution.
- Do not update long-term project knowledge with unverified claims.
- Do not delete or clean up work evidence before archive is closed.
