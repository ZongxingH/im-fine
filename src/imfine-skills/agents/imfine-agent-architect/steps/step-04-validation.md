# Step 04: Architecture Validation

Goal: confirm the design is ready for imfine Task Planner and native Agent execution.

Checklist:

- Every required behavior has an architecture path.
- Every risky decision has a mitigation or explicit blocker.
- Every Agent-conflict point has an implementation rule.
- Every proposed parallel boundary has non-overlapping write scope.
- Verification commands are concrete.
- Acceptance matrix inputs can be authored by Agents.
- Archive and project knowledge updates are not forgotten.

If any item fails, return to the relevant step. If the failure is due to missing requirement context, hand off to Intake or Product Planner instead of guessing.
