# Step 02: Architecture Decisions

Goal: make the decisions that independent Agents must implement consistently.

Decision categories:

- data model or persistence boundary
- API and integration contract
- frontend state and routing boundary
- backend service/module ownership
- test strategy and runtime requirements
- error handling and security posture
- compatibility, migration, or release constraints

For each decision, write:

- decision
- alternatives considered
- evidence
- implementation rule for Agents
- risk if omitted

Do not decide by preference alone. Brownfield decisions must cite observed project patterns.
